Gmail自動仕分けシステム 設計書
合同会社圓子水産 — 最終更新: 2026-04-02
A. システム概要

受信メールをルールベース＋AI分類のハイブリッドで自動仕分けするシステム。Discordで双方向操作（確認・修正・返信）が可能。ユーザーのフィードバックから新ルールを自動学習する。

  • 5分間隔で受信トレイをスキャンし、117ルールと照合してラベルを自動付与
  • マッチ条件: from（送信元）+ subject（件名）+ body（本文snippet ~200文字）
  • confidence high → ラベル付与＋アーカイブ、low → 受信トレイに残す
  • 安全ガード: 件名＋本文に重要キーワード（至急・クレーム等）を検知 → 受信トレイに残してDiscord即通知
  • 未マッチメール → AI分類（Claude Code）→ Discord #gmail-ai で確認（✅❌✏️💬）
  • 自動学習: フィードバックのsnippetを分析 → 同ドメイン複数ラベル時はbody条件を自動抽出 → 2回確認でルール昇格

D. ルール仕様
  • ルールはJSONで定義（現在117ルール）
  • 条件: from（送信元の部分一致）、subject（件名キーワードの部分一致）、body（本文snippet ~200文字の部分一致）
  • 複数条件はAND、各条件内の配列はOR
  • マッチ優先度: 条件キー数が多いルール（より具体的）が先に評価
  • confidence: "high" → ラベル付与＋アーカイブ、"low" → ラベルのみ（受信トレイに残す）
  • important: true → Discord #一般に通知
// ルール例1: from + subject { "name": "食べチョク注文", "conditions": { "from": ["tabechoku.com"], "subject": ["受注", "注文", "要確認"] }, "label": "通販/食べチョク", "archive": true, "important": false, "confidence": "high" }
// ルール例2: from + body（本文で区別） { "name": "IVRy請求関連", "conditions": { "from": ["ivry.jp"], "body": ["請求", "料金", "お支払い"] }, "label": "経理", "archive": true, "important": false, "confidence": "high" } // 同じivry.jpからbody条件なしの別ルール → 経理以外は総務/Ivryへ
body条件はGmail APIのsnippet（本文冒頭~200文字）と照合。追加APIコールなし。
E. 安全ガード

ルール未マッチのメールに対して、件名＋本文snippet＋送信元を安全キーワードでスキャン。検知したメールは受信トレイに残し、Discordで即時通知する。

  • キーワードガード: 至急、緊急、クレーム、返金、返品、督促、法的、訴訟、解約、事故、苦情、催促、未払い、滞納、期限切れ、違反、警告
  • 初回ドメインガード: 過去30日のログに未出現ドメイン → 強制 confidence: "low"
  • 信頼済みドメイン除外: rules/marukosuisan.json の from パターンに含まれるドメインはガード対象外
安全ガードは件名だけでなく本文snippetもスキャン。件名が普通でも本文に「至急」があれば検知される。
F. AI分類 + Discord双方向操作

ルール未マッチかつ安全ガード通過のメールは、Claude Code（/loop 5分間隔）がAI分類し、Discord #gmail-aiに投稿する。ユーザーはリアクションで操作。

リアクション動作後続処理
✅分類を確認corrections.jsonに記録（snippet付き）→ ルール昇格判定
❌分類を否定corrections.jsonに記録 → ✏️で正しいラベルを選択
✏️ラベル修正ラベル一覧を表示 → 番号で選択 → corrections.jsonに記録 → ルール昇格判定
💬返信リクエストpending-replies.json → Claude Codeが返信案生成 → Discordに投稿 → 承認で送信

G. 学習の仕組み

2つの経路で学習し、body条件を含むルールを自動生成する。

経路1: Discordフィードバック → リアルタイム学習

  1. ユーザーがDiscordで✅✏️でフィードバック
  2. corrections.jsonにsnippet付きで記録
  3. 同ドメイン+同ラベルが2回確認 → ルール自動昇格
  4. 同ドメインで複数ラベルがある場合 → snippetからbody条件を自動抽出して区別

経路2: learner.js → 1日1回バッチ学習

  1. ユーザーがGmailで手動でラベル付け＋アーカイブ
  2. learnerが直近48時間の手動仕分けをスキャン（snippet付き）
  3. ドメイン＋ラベルのパターン＋snippetキーワードを分析
  4. 同ドメインが複数ラベルに分かれている → body条件付き候補ルールを生成
  5. Discordに候補通知 → Claude Codeで承認・修正
body条件の自動抽出: 同じドメインが複数ラベルに振り分けられた時、各ラベルのsnippetを比較して「このラベルにしかない特徴的なキーワード」を抽出する。
H. 安全設計
  • gmail-sorter/processed ラベルで二重処理防止
  • confidence low のメールは受信トレイに残す（見落とし防止）
  • 安全キーワード検知メールはアーカイブされない（至急・返金等）
  • 初回ドメインは強制 low confidence（初見メールを見逃さない）
  • learner候補は自動適用されない（明示的承認が必要）
  • --dry-run で実行前に何が起きるか確認可能（複数マッチ警告付き）
  • uncaughtException / unhandledRejection をキャッチしてDiscord通知
  • ルールファイルの書き込みはアトミック（tmp → rename）
  • ai-notifications.jsonは200件上限で自動クリーンアップ
