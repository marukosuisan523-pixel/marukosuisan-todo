記憶システム（2層構造）
mempalace + engram + md の3システム統合
稼働中
何ができる？
会話の内容・決定事項・作業記録をセッションを超えて記憶。ベクトル類似度検索+キーワード検索+ナレッジグラフで過去の情報を引き出せる
2層構造
上の階（要約・構造化）: wing/room に手動保存（add_drawer）。高品質な決定・設定・方針を格納
下の階（毎ターン自動ログ）: wing/room/log に全応答を自動保存（Stopフック）。漏れゼロの生データ
mempalace
メイン記憶。ChromaDBベクトル検索 + SQLiteナレッジグラフ（時間軸クエリ対応）。19,600件超、751MB
engram
FTS5キーワード検索インデックス。add_drawer時にフックで自動コピー。8MB
md (shared-memory)
セッション起動時に自動読込される永続メモリ。119ファイル、2.1MB
自動フロー
保存: add_drawer → mempalace上の階 → [hook] → engram FTS5
自動ログ: 毎ターン → mempalace下の階（room/log）
検索: 毎プロンプトでengram FTS5 + mempalaceベクトル検索 → コンテキスト自動注入
起動時: L0 identity.txt自動表示 + diary_readで前回要約確認
ナレッジグラフ
2,864エンティティ / 5,569トリプル。kg_query（as_of=日付で時間軸フィルター）、kg_timeline（変遷表示）
使用部門
全社共通（秘書室）
技術
mempalace v3.1.0 MCP + engram v1.11.0 MCP + Claude Code auto-memory
保存はadd_drawer一本に統一。mem_saveは使わない。セッション終了前にdiary_writeで要約日記を必ず書く
