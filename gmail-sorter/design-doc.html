合同会社圓子水産 — 最終更新: 2026-03-21
A. システム概要

受信メールをルールに基づいて自動でラベル付け＋アーカイブするシステム。手動仕分けから新ルールを学習する仕組み付き。PCがログインしていれば常時稼働（Claude不要）。

  • 5分間隔で受信トレイをスキャンし、53ルールと照合してラベルを自動付与
  • confidence high のメールはラベル付与＋アーカイブ、low は受信トレイに残す
  • 重要メール・未分類・エラーを Discord Bot で通知
  • learner が手動仕分けを1日1回スキャン → 候補を Discord に通知 → 承認で正式ルール化

D. ルール仕様
  • ルールはJSONで定義
  • 条件: from（ドメインの部分一致）、subject（キーワードの部分一致）
  • 複数条件はAND、各条件内の配列はOR
  • マッチ優先度: 条件キー数が多いルール（より具体的）が先に評価
  • confidence: "high" → ラベル付与＋アーカイブ、"low" → ラベルのみ
  • important: true → Discord通知を送る
// ルール例 { "name": "食べチョク注文", "conditions": { "from": ["tabechoku.com"], "subject": ["受注", "注文", "要確認"] }, "label": "通販/食べチョク", "archive": true, "important": false, "confidence": "high" }
E. 学習（learner）の仕組み
  1. ユーザーがGmailで手動でラベル付け＋アーカイブ
  2. learnerが1日1回スキャン（直近48時間の手動仕分けを検出）
  3. ドメイン＋ラベルのパターンを分析
  4. 既存ルールにないパターンを候補としてJSONに保存
  5. Discordに候補通知
  6. Claude Codeで承認作業（候補の内容を確認 → 修正 → 正式ルールに昇格）

G. 安全設計
  • gmail-sorter/processed ラベルで二重処理防止
  • confidence low のメールは受信トレイに残す（見落とし防止）
  • learner候補は自動適用されない（明示的承認が必要）
  • --dry-run で実行前に何が起きるか確認可能（複数マッチ警告付き）
  • uncaughtException / unhandledRejection をキャッチしてDiscord通知
  • ルールファイルの書き込みはアトミック（tmp → rename）
