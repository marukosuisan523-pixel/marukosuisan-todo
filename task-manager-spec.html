システム概要

Discordをメインの入り口として、Claude Codeが頭脳となりタスク管理・業務指示・リマインドを一元管理するシステム。

全体フロー

Discord（瑞樹さん）
  |
  v
Claude Code（頭脳）
  |--- 即実行可能 --> 既存スキルで実行 --> Discord返信
  |--- スタッフ指示+日時 --> Googleカレンダー登録 --> 10分後TODOボード自動同期
  |--- 業務タスク --> GAS API --> TODOボード + LINE通知
  |--- 個人タスク --> ローカル保存 --> Discordリマインドのみ
  |--- 質問/相談 --> 通常回答

スタッフ側：
  TODOボード（Web）で確認 / LINE通知で受け取り / Googleカレンダーで確認

タスクの分類ルール

条件行き先通知例
スタッフ指示 + 日時ありGoogleカレンダーカレンダー通知 + 10分後TODOボード同期「小川に明日の配送準備やっといて」
業務タスク（スタッフ共有）TODOボードTODOボード + LINE通知「小川に発注リスト確認させて」
個人タスクローカル保存Discordリマインドのみ「モバイル充電器買う」
即実行可能既存スキルDiscord返信「今月の給与処理して」

拡張機能

1. サブタスク分解

大きなタスクを小さなステップに分解。それぞれに期限を設定し、サブタスクの期限でもリマインドが発火する。全サブタスク完了で親タスクも自動完了。

消費税の申告（期限: 3/31）
  ├ 必要書類を集める（3/25）
  ├ 税理士に確認（3/27）
  └ e-Taxで申告提出（3/31）

2. 着手日（start_date）

期限だけでなく「いつから始めるか」も管理。期限が1週間以上先のタスクには着手日を自動提案（期限の3日前 or 半分の時点）。着手日の朝に「今日から着手しましょう」とリマインド。

3. 朝のプランニング（毎朝5:00）

その日のアクションを提案する朝のダイジェスト。

🌅 おはようございます！今日のプラン

📌 今日が期限
・スマートモニター買う

🚀 今日から着手
・消費税の申告 → まず「書類を集める」から

💡 そろそろ着手しませんか？
・就業規則修正（3/31期限、残り11日）

⚠ 期限切れ
・（なし）

4. 夕方の進捗チェック（18:00）

今日の期限・着手予定だったタスクについて進捗を確認。返事がなくても催促はしない（朝のリマインドに任せる）。

5. タスク依存関係（blocked_by）

タスク間の順番を管理。ブロッカーが完了するまでリマインドを抑制し、完了時に「次のタスクに着手できます」と通知。

北洋銀行：手続き
  └ [完了後] → 給与振り込みの続き（ブロック解除）

タイミング内容

GAS API仕様

アクションメソッド用途
getAllGET全タスク・社員リスト取得
addTodoPOST単一タスク追加（競合安全・LINE通知自動発火）
updateTodoPOST単一タスク更新・完了（カレンダー✅・LINE通知自動）
setAllPOST全件上書き（TODOボードUI用・Claude Codeは使用しない）
completeTodoPOSTカレンダーイベント完了マーク
